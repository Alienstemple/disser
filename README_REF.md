## 2. Назначение программы

Система предназначена для автоматического обнаружения и нейтрализации DDoS-атак в компьютерных сетях в реальном времени. Программа анализирует сетевой трафик, выявляет аномалии, классифицирует тип атаки и автоматически применяет правила защиты через фаервол.

### Основные цели:
- Раннее обнаружение DDoS-атак с помощью нейронных сетей
- Автоматическая генерация правил защиты через языковую модель (LLM)
- Минимизация ложных срабатываний благодаря многоуровневой проверке
- Адаптивная защита с автоматическим откатом правил

## 4. Функциональные возможности

### 4.1 Модуль обнаружения (detection/)
- Захват сетевого трафика в реальном времени (pcap/netflow)
- Извлечение 15 признаков из каждого пакета
- Классификация трафика с помощью CNN+LSTM нейронной сети
- Порог срабатывания: 0.85 (настраивается)
- Время обнаружения: < 100 мс

### 4.2 Модуль анализа (analysis/)
- Анализ IP-источников трафика
- Вычисление репутации IP-адресов
- Группировка по подсетям (/24, /16)
- Классификация типа атаки по сигнатурам
- Сбор метрик эффективности (Precision, Recall, F1, MCC)

### 4.3 LLM-модуль (llm/)
- Генерация промптов на основе контекста атаки
- Поддержка провайдеров: OpenAI, Groq, Ollama, Mock
- Парсинг ответов и извлечение правил фаервола
- Валидация сгенерированных правил

### 4.4 Модуль нейтрализации (mitigation/)
- Управление правилами фаервола (iptables/nftables/pf)
- Блокировка IP-адресов и портов
- Rate limiting для подозрительных источников
- Автоматический откат правил по TTL
- Откат по ложным срабатываниям
- Градуированный откат (DROP → RATE_LIMIT → REMOVE)

### 4.5 Модуль логирования (logger/)
- Логирование событий безопасности (JSON/text)
- Вращение файлов логов
- Экспорт метрик в формате Prometheus
- Временные ряды для анализа трендов

### 4.6 Модуль конфигурации (config/)
- Типизированная конфигурация (YAML)
- Валидация настроек при загрузке
- Управление белыми/чёрными списками
- Поддержка временных правил (TTL)

## 6. Особенности реализации

### 6.1 Архитектурные решения

**Модульная архитектура:**
```
src/
├── detection/     → Обнаружение атак
├── analysis/      → Анализ трафика
├── llm/           → Генерация правил
├── mitigation/    → Применение правил
├── logger/        → Логирование
├── config/        → Конфигурация
└── main.py        → Точка входа
```

**Преимущества:**
- Независимое тестирование модулей
- Лёгкое расширение функциональности
- Возможность отключения компонентов

### 6.2 CNN+LSTM архитектура

```
Вход: (batch, timesteps=60, features=15)
│
├── Conv1D(64, kernel=3) + MaxPooling + Dropout
├── Conv1D(128, kernel=3) + MaxPooling + Dropout
├── LSTM(128) + Dropout
├── Dense(64) + Dropout
└── Dense(1, sigmoid) → attack_probability
```

**Обоснование:**
- CNN выявляет локальные паттерны (всплеск SYN-пакетов)
- LSTM запоминает долгосрочные зависимости (нарастание нагрузки)

### 6.3 LLM-генерация правил

**Поток данных:**
```
Атака → Prompt Builder → LLM API → Response Parser → Firewall Rules
```

**Пример промпта:**
```
Ты — эксперт по кибербезопасности.
Тип атаки: SYN Flood
Подозрительные IP: 192.168.1.100, 10.0.0.50
Целевые порты: 80, 443
Сгенерируй правила iptables для блокировки.
```

### 6.4 Механизм отката правил

**Типы откатов:**
1. **TTL expired** — автоматический откат по времени
2. **False positive** — откат при жалобах пользователей
3. **Graduated rollback** — постепенное ослабление
4. **Manual** — ручной откат администратором

**Градуированный откат:**
```
DROP → RATE_LIMIT(5/sec) → RATE_LIMIT(20/sec) → REMOVE
```

### 6.5 Обработка ложных срабатываний

**Пороги для отката:**
| Уровень | FP-отчётов | Действие |
|---------|------------|----------|
| Low | 3 | Откат правила |
| Medium | 5 | Откат + запись в лог |
| High | 10 | Откат + уведомление |

## 7. Установка и запуск

### 7.1 Требования
- Python 3.10+
- Linux/macOS
- root-права (для работы с фаерволом)

### 7.2 Установка зависимостей
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 7.3 Запуск
```bash
# Тестовый режим (без реальных правил)
python main.py --config config.yaml --dry-run --verbose

# Продуктовый режим
python main.py --config config.yaml
```

### 7.4 Конфигурация
Основные настройки в `config.yaml`:
```yaml
detection:
  attack_probability_threshold: 0.85
  window_size_sec: 10

llm:
  provider: mock  # openai / groq / ollama

mitigation:
  firewall_backend: mock  # iptables / nftables / pf
  dry_run: true
```

## 10. Ограничения

1. **Требует root-прав** для работы с фаерволом
2. **Mock-режим по умолчанию** — требуется настройка для production
3. **Зависимость от LLM** — требует API ключ или локальную модель
4. **Не защищает от L7-атак** сложного уровня (требует WAF)

### B. Примеры использования

**Блокировка IP:**
```python
from mitigation import FirewallController, FirewallBackend

controller = FirewallController(backend=FirewallBackend.IPTABLES)
controller.block_ip("192.168.1.100", ttl_seconds=3600)
```

**Анализ трафика:**
```python
from analysis import IPAnalyzer

analyzer = IPAnalyzer()
analyzer.analyze_packets(packets)
suspicious = analyzer.get_suspicious_ips(top_n=10)
```

**Генерация правил через LLM:**
```python
from llm import PromptBuilder, LLMClient, ResponseParser

builder = PromptBuilder(firewall_backend="iptables")
prompt = builder.build_prompt(attack_context)

client = LLMClient(provider="mock")
response = client.generate(prompt)

parser = ResponseParser()
rules, explanation = parser.parse(response.content)
```
