
Модуль обнаружения атаки (CNN + LSTM)

Алгоритм работы:
1. Запустить сбор трафика (pcap / netflow).
2. Каждые N секунд:
   a. Извлечь признаки за окно [t-N, t].
   b. Подать тензор в CNN+LSTM.
   c. Если attack_prob > threshold:
        i.   Определить тип атаки по признакам.
        ii.  Выделить подозрительные IP.
        iii. Сформировать промпт для LLM.
        iv.  Отправить запрос в LLM.
        v.   Распарсить ответ → получить правила.
        vi.  Применить правила через firewall_controller.
        vii. Залогировать событие.
   d. Иначе: продолжать мониторинг.
3. Параллельно:
   - Собирать метрики эффективности.
   - Анализировать ложные срабатывания.
   - Автоматически откатывать старые правила.

Конфигурация системы (задаем в config-файле)
Порог срабатывания модели (attack_probability_threshold = 0.85)
Интервал анализа (window_size_sec = 10)
Белый список IP (не блокировать никогда)
Целевые порты/сервисы (например, HTTPS на 443)

Модель принимает временной ряд признаков, полученный из сетевого трафика за фиксированное окно (например, 60 секунд). Каждая временная метка содержит вектор признаков.


Архитектура нейросети (по слоям)
# CNN: выявляет локальные аномалии в признаках (например, всплеск мелких пакетов)
Conv1D(filters=64, kernel_size=3, activation='relu')
MaxPooling1D(pool_size=2)

# LSTM: моделирует долгосрочные зависимости (например, нарастание нагрузки)
LSTM(units=128, return_sequences=False)
Dropout(0.3)

# Классификатор
Dense(64, activation='relu')
Dense(1, activation='sigmoid')  # Вероятность атаки

 Выход модели

attack_probability: float ∈ [0, 1]
Если attack_probability > threshold (например, 0.85):
Система считает, что идёт DDoS-атака
Запускается анализ источников → выделение подозрительных IP
Передача управления модулю нейтрализации
